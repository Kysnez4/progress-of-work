## Проект 1: Habit Tracker API — Трекер полезных привычек

**Сервис для формирования полезных привычек с Telegram-уведомлениями**

### 📌 О проекте
REST API сервис, помогающий пользователям формировать полезные привычки по методике Джеймса Клира ("Атомные привычки"). Пользователи могут создавать привычки, отслеживать их выполнение и получать напоминания в Telegram.

### 🛠️ Технологии
- **Backend**: Django 4.2, Django REST Framework
- **База данных**: PostgreSQL
- **Аутентификация**: JWT (djangorestframework-simplejwt)
- **Фоновые задачи**: Celery, Redis
- **Документация**: drf-yasg (Swagger, ReDoc)
- **Интеграции**: Telegram Bot API
- **Тестирование**: Pytest

### ✨ Функциональность
- Регистрация и JWT-авторизация пользователей
- CRUD операции с привычками
- Просмотр публичных привычек других пользователей
- Валидация правил формирования привычек (периодичность 1-7 дней, время выполнения ≤120 сек и др.)
- Пагинация (5 привычек на страницу)
- Разграничение прав доступа
- Автоматические Telegram-уведомления о необходимости выполнить привычку
- Полная документация API (Swagger UI)

## Проект 2: Django + Docker + Celery + Redis + PostgreSQL

**Готовый шаблон для развертывания Django проекта в Docker-контейнерах**

### 📌 О проекте
Полноценная конфигурация для контейнеризации Django приложения с использованием Docker Compose. Включает настройку всех необходимых сервисов: PostgreSQL, Redis, Celery worker и Celery beat для периодических задач. Проект развернут на удаленном сервере и доступен по публичному адресу.

### 🛠️ Технологии
- **Контейнеризация**: Docker, Docker Compose
- **Backend**: Django, Gunicorn
- **База данных**: PostgreSQL
- **Очереди задач**: Celery, Redis
- **CI/CD**: GitHub Actions (автоматический деплой)
- **Сервер**: Ubuntu, Nginx (опционально)

### ✨ Функциональность
- Полная контейнеризация Django проекта
- Разделение на сервисы: web, db, redis, celery, celery-beat
- Автоматическое применение миграций при запуске
- Сборка статических файлов
- Переменные окружения через .env файл
- CI/CD pipeline для автоматического деплоя на сервер
- Инструкции по бэкапу БД и устранению неполадок
